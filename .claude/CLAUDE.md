# プロジェクト固有の指示

## Gitコミットメッセージ

コミットメッセージは必ず日本語で記述すること。

### コミットメッセージの形式
- タイトル行：変更内容を簡潔に日本語で記述
- 空行
- 詳細：変更の詳細を箇条書きで日本語で記述
- 空行
- フッター：Claude Codeの署名

### 例
```
機能Xを追加

- コンポーネントYを作成
- 関数Zを実装
- テストを追加

🤖 Generated with [Claude Code](https://claude.com/claude-code)

Co-Authored-By: Claude <noreply@anthropic.com>
```

## コード編集後のエラーチェック

コードの編集や新規ファイルの作成を行った後は、必ず以下のエラーチェックを実施すること。

### 1. 診断エラーの確認
`mcp__ide__getDiagnostics`ツールを使用して、編集したファイルにeslintエラーやTypeScriptエラーが発生していないか確認する。

### 2. エラーが発生した場合
エラーが発見された場合は、そのエラーを解消するまで作業を完了としない。以下の手順でエラーを解消すること。

- エラーの内容を分析する
- 該当箇所を修正する
- 再度診断を実行し、エラーが解消されたことを確認する

### 3. 確認するエラーの種類
- eslintエラー
- TypeScriptの型エラー
- モジュール解決エラー
- その他のコンパイルエラー

### 4. 実施タイミング
- ファイルの編集後
- 新規ファイルの作成後
- 複数ファイルを編集した場合は、全ての編集完了後

この手順を省略せず、必ず実施すること。
